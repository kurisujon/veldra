    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

## 2. Logic Chain

1. The database migration `20260622000000_phase4_documents.sql` dictates the exact arguments required by the database functions and table columns. Thus, the validation logic in `validation.ts` must perfectly match these constraints (validating `caseId` as UUID, restriction of `type` to `'PSABirth' | 'PSAMarriage' | 'TOR' | 'SF10' | 'Diploma'`).
2. Server actions must resolve cookies and maintain authentication context. Using `createClient()` from `@/lib/supabase/server` satisfies this requirement, automatically passing the user's JWT to Supabase backend API calls and RPCs.
3. In `uploadDocument`, files must first be written to storage to ensure we have a path and file attributes for the database row. However, if the database record insertion fails later, the storage file becomes orphaned. Therefore, we require an explicit try-catch structure that deletes the uploaded storage object on RPC failure.
4. In `deleteDocument`, the database-level `delete_document` RPC handles both the removal of `storage.objects` metadata rows and the deletion of the `documents` record in a single database transaction. This ensures referential integrity.

---

## 3. Caveats

- Node.js global environment context must support standard `File` and `Blob` APIs (Node 18+). Since Veldra utilizes Next.js 14, this is standard.
- We assume the `documents` private bucket has been successfully provisioned by the database migration (Milestone 2). If not, storage operations will throw errors.
- The `delete_document` RPC does not delete the physical file if the row deletion is somehow blocked by database-level triggers, though standard Supabase operation cascades this deletion immediately.

## 4. Conclusion

The designed codebase structure (`types.ts`, `validation.ts`, and `actions.ts`) provides a complete, robust, and type-safe implementation path for Phase 4 Milestone 3. The server actions strictly conform to the interface contracts defined in `PROJECT.md` and enforce non-bypassable backend verification.
