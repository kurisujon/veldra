# Copy Guidelines

## Language & Tone
- **Simple & Direct**: Write copy that is easily understandable by HR, document staff, and developers.
- **Professional**: Maintain a respectful, neutral tone.

## Prohibited Elements
- **NO Marketing Buzzwords**: Avoid terms like "synergy", "revolutionize", "next-gen".
- **NO Excessive AI Terminology**: Do not use "AI Magic", "Neural Processing", or "Smart Brain". Instead use "Automated Check", "Extraction", or "Analysis".
- **NO Overly Technical Language**: Avoid exposing database terms, exact API responses, or raw JSON structures to the end user.

## Examples
- *Instead of:* "Our cutting-edge AI neural network is magically reading your files."
- *Use:* "Extracting document data..."

- *Instead of:* "Validation Error: Regex mismatch on Date Field."
- *Use:* "Please ensure the date is in MM/DD/YYYY format."
